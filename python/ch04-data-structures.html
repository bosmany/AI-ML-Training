Chapter 4:
Data Structures

Real datasets don't arrive as five neat variables — they arrive as thousands of records, each with several fields, often nested inside each other. This chapter covers the four containers Python gives you to hold that data: lists, tuples, dicts, and sets — and the shape you'll see constantly once you start loading real data: lists of dictionaries.

🕐 ~55 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
4.1 — Lists in Depth: Mutable, Ordered Collections

You've used lists since Chapter 1. A list is mutable — you can change it after creating it — and ordered — items keep the position you put them in. Its methods change the list in place and usually return None, which is a common source of confusion.

List methods — building and reordering a scores list
scores = [72, 85, 60, 91, 78] scores.append(95) # adds to the end — modifies scores in place print(scores) scores.remove(60) # removes the FIRST 60 it finds print(scores) scores.sort(reverse=True) # sorts in place, highest first print(scores) print(f'Top score: {scores[0]}') last = scores.pop() # removes AND returns the last item print(f'Removed: {last}, remaining: {scores}')
⚠️
Beginner trap: scores = scores.sort() sets scores to None! .sort() reorders the list in place and returns nothing. Just call scores.sort() on its own line — don't reassign it.
4.2 — Tuples: Immutable, Ordered Collections

A tuple looks like a list but uses () instead of [], and once created, it cannot be changed. Use a tuple when a value shouldn't ever be modified — coordinates, a fixed record, a return value made of several parts.

Tuples and unpacking
point = (3, 4) x, y = point # unpacking — x=3, y=4 print(f'x={x}, y={y}') # point[0] = 5 # <-- TypeError! tuples don't support item assignment # Unpacking a list of tuples in a loop — a very common ML pattern records = [('alice', 28, 'Hubli'), ('bob', 34, 'Pune')] for name, age, city in records: print(f'{name} is {age} from {city}')
💡
Why this matters in ML: a dataset row is often naturally a tuple — fixed fields, in a fixed order, that shouldn't change once loaded. Functions that return multiple values (like train_test_split() in sklearn) return them as a tuple.
4.3 — Dictionaries: Key-Value Pairs

A dict maps keys to values. It's the single most-used data structure in real ML code — model configs, API responses, and JSON files are all, fundamentally, dicts.

Building and reading a model config
model_config = {'name': 'ResNet50', 'epochs': 50, 'lr': 0.001} print(model_config['epochs']) # 50 — direct access by key print(model_config.get('batch_size', 32)) # 32 — .get() returns a default instead of crashing # model_config['batch_size'] # <-- this would raise a KeyError! the key doesn't exist yet model_config['batch_size'] = 32 # adding a new key is just an assignment for key, value in model_config.items(): print(f'{key}: {value}')
⚠️
Real bug that kills real ML pipelines: config['batch_size'] raises KeyError the instant that key is missing — and it WILL be missing sometimes, from an incomplete config file or an old dataset version. config.get('batch_size', 32) never crashes; it just uses your default.
4.4 — Sets: Unique, Unordered Collections

A set holds only unique values, with no guaranteed order, and supports fast mathematical operations: union, intersection, difference. Use it to deduplicate, or to compare two collections of things.

Comparing feature sets between two model versions
model_a_features = {'age', 'income', 'education', 'city'} model_b_features = {'age', 'income', 'job_title'} common = model_a_features & model_b_features # intersection — in BOTH only_a = model_a_features - model_b_features # difference — in A but not B combined = model_a_features | model_b_features # union — in EITHER # sorted() gives a stable, printable order — sets themselves have none print(f'Common: {sorted(common)}') print(f'Only in A: {sorted(only_a)}') print(f'All features: {sorted(combined)}') # A classic use: removing duplicate labels raw_labels = ['cat', 'dog', 'cat', 'bird', 'dog', 'cat'] unique_labels = set(raw_labels) print(f'Unique classes: {sorted(unique_labels)}')
4.5 — The Shape of Real Data: Lists of Dictionaries

This is the pattern you'll see everywhere once you start loading real datasets — CSV rows, API responses, JSON files. Each record is a dict, and the whole dataset is a list of them.

A tiny dataset, ML-shaped
candidates = [ {'name': 'Alice', 'skills': ['Python', 'SQL'], 'score': 0.92}, {'name': 'Bob', 'skills': ['Java'], 'score': 0.75}, ] for c in candidates: print(f"{c['name']}: {c['score']:.0%} — skills: {', '.join(c['skills'])}")
4.6 — Sorting Complex Structures

You already met sorted(..., key=lambda ...) in Chapter 3. It works exactly the same way on a list of dicts — the lambda just picks which field to sort by.

Ranking candidates by score
candidates = [ {'name': 'Alice', 'score': 0.92}, {'name': 'Bob', 'score': 0.75}, {'name': 'Carol', 'score': 0.81}, ] ranked = sorted(candidates, key=lambda c: c['score'], reverse=True) for i, c in enumerate(ranked, start=1): print(f'#{i}: {c["name"]} — {c["score"]:.0%}')
⭐
enumerate(ranked, start=1) — the optional start argument you haven't seen yet lets ranks begin at 1 instead of 0. Small thing, very commonly used for leaderboards and reports.
4.7 — Nested Dicts: The JSON Shape

Config files and API responses often nest a dict inside a dict. Accessing a nested value just means chaining the keys: data['train']['accuracy'].

A nested dataset-stats structure
dataset_stats = { 'train': {'size': 8000, 'accuracy': 0.91}, 'val': {'size': 1000, 'accuracy': 0.87}, 'test': {'size': 1000, 'accuracy': 0.85}, } for split, stats in dataset_stats.items(): print(f"{split}: {stats['size']} samples, {stats['accuracy']:.0%} accuracy") # Direct nested access — chain the keys: print(f"Test accuracy specifically: {dataset_stats['test']['accuracy']:.0%}")
Chapter 4 Summary
  • Lists are mutable and ordered. Methods like .sort() and .append() change the list in place and return None — never reassign their result.
  • Tuples are immutable and ordered — use them for fixed records, and for unpacking multiple values at once.
  • Dicts map keys to values. Use .get(key, default) instead of [key] whenever the key might not exist.
  • Sets hold unique values and support & (intersection), - (difference), | (union).
  • Lists of dicts are the shape of real datasets. Nested dicts are the shape of real configs and JSON.
  • sorted(data, key=lambda x: x['field']) ranks any list of dicts by any field.
