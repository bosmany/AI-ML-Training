Chapter 1:
Python Basics

Before you train your first model, build your first neural network, or deploy your first AI — you need Python. This chapter gives you the absolute foundation: how Python stores data, how to manipulate it, and how to format output. Every concept here will appear in real ML code.

🕐 ~45 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
1.1 — What Is Python and Why Do AI/ML Engineers Use It?

Python is a programming language created in 1991. Today it is the dominant language for AI and ML — not because it is the fastest (it isn't), but because:

  • The libraries are unmatched. NumPy, Pandas, PyTorch, TensorFlow, scikit-learn, HuggingFace — every major ML tool is built for Python first.
  • It reads like English. if accuracy > 0.9: is obvious even without programming experience. This matters when you are learning complex ML concepts — you want the language to stay out of the way.
  • It is what your future colleagues use. Every ML team at Google, Meta, and every AI startup writes Python. Learning it is not optional.
✅
Your goal for this chapter is not to memorize syntax. It is to understand how Python thinks about data — so that when you write accuracy = 0.94 next month in a PyTorch training loop, you know exactly what is happening.
1.2 — Variables: How Python Stores Data

A variable is a name that points to a value. When you write loss = 0.2341, Python creates a box labelled loss and puts the value 0.2341 inside it. Any time you use loss later, Python looks it up and gives you 0.2341.

Unlike some other languages, Python is dynamically typed — you never need to declare what type of data a variable holds. Python figures it out by looking at the value.

Variables in ML — run this
# In a real ML training loop, you would see variables like these: model_name = 'ResNet50' # str — name of the model epoch = 1 # int — which training pass we are on loss = 0.2341 # float — how wrong the model is right now accuracy = 0.9423 # float — how correct (as a fraction) is_training = True # bool — are we training or evaluating? # Python tells you what type a variable is: print(type(model_name)) # <class 'str'> print(type(epoch)) # <class 'int'> print(type(loss)) # <class 'float'> print(type(is_training)) # <class 'bool'> # You can reassign any variable any time: epoch = 2 # epoch was 1, now it is 2 loss = 0.1987 # loss dropped — model is improving print(f'Epoch {epoch}: loss improved to {loss}')
💡
Naming convention: Python variables use snake_case — words separated by underscores. learning_rate, not LearningRate or learningRate. This is the standard across all Python ML code.
1.3 — The Four Core Data Types

Python has four data types you will use constantly in ML. Understanding each one — and when to use which — prevents a whole class of bugs.

int
Whole numbers. No decimal point.
epoch = 50
batch_size = 32
num_layers = 12
ML use: counting things — epochs, layers, samples
float
Decimal numbers. Always has a point.
lr = 0.001
loss = 0.2341
dropout = 0.5
ML use: model weights, loss values, accuracy
str
Text. Always in quotes (single or double).
model = 'BERT'
path = '/models/v2'
label = "cat"
ML use: file paths, class names, model names
bool
Only True or False. Capital first letter.
is_training = True
verbose = False
use_gpu = True
ML use: flags, mode switches, conditions
All four types in action
epochs = 50 # int — whole number learning_rate = 0.001 # float — decimal model_name = 'ResNet50' # str — text in quotes use_gpu = True # bool — only True or False # Check each type: print(epochs, type(epochs)) print(learning_rate, type(learning_rate)) print(model_name, type(model_name)) print(use_gpu, type(use_gpu)) # Important: '50' is NOT the same as 50 a = '50' # str b = 50 # int print(a == b) # False! Different types
1.4 — Type Conversion: The #1 Beginner Bug in ML

When you load a CSV file in Python, every single value comes in as a string — even numbers. This catches almost every beginner off guard. You cannot add strings. You cannot calculate with them. You must convert them first.

⚠️
Real bug that kills real ML projects: You load candidate ages from a CSV. They come as strings: '25', '30', '45'. You try to calculate average age. Python gives you a TypeError. Your entire preprocessing pipeline crashes. This is type conversion — learn it now.
Type conversion — every ML engineer uses this daily
# ── Converting TO int ────────────────────────────────── age_str = '25' # came from CSV — it's a STRING age_int = int(age_str) # now it's an integer: 25 print(age_int + 5) # 30 — works now # ── Converting TO float ──────────────────────────────── score_str = '94.5' score_float = float(score_str) # 94.5 print(score_float / 100) # 0.945 # ── The TRAP: int('3.9') CRASHES ────────────────────── # int('3.9') <-- DO NOT do this. Python will crash. # The correct way: go through float first safe = int(float('3.9')) # step 1: float('3.9')=3.9 step 2: int(3.9)=3 print(safe) # 3 (truncates, does NOT round) # ── Converting TO str ────────────────────────────────── loss = 0.2341 message = 'Loss is: ' + str(loss) # must convert to combine print(message) # ── bool conversions ─────────────────────────────────── print(bool(0)) # False — zero is falsy print(bool(1)) # True — any nonzero is truthy print(bool('')) # False — empty string is falsy print(bool('hi')) # True — non-empty string is truthy

1.5 — Strings: Working With Text Data

Strings are how Python handles text. In ML, you use strings constantly: loading file paths, labelling classes, printing progress messages, processing text in NLP models. Strings have a rich set of built-in methods that let you clean, transform, and analyze text.

String methods you will use in every ML project
text = ' Hello, World! This is Python for ML. ' # Cleaning methods — NLP preprocessing uses all of these print(text.strip()) # remove leading/trailing spaces print(text.lower()) # lowercase — always do this before NLP print(text.upper()) # uppercase print(text.strip().lower()) # chain methods together # Finding and replacing print(text.replace('Python', 'Python3')) # replace print(text.find('ML')) # index of first match print('Python' in text) # True if present # Splitting — tokenization at its simplest words = text.strip().split() # split on whitespace print(words) print(len(words), 'words') csv_line = 'Alice,28,ML Engineer,Hubli' parts = csv_line.split(',') # split on comma — reading CSV print(parts) # String slicing — accessing characters by index s = 'Python for ML' print(s[0]) # P — index starts at 0 print(s[-1]) # L — negative index counts from end print(s[0:6]) # Python — characters 0,1,2,3,4,5 print(s[7:]) # for ML — from index 7 to end print(s[::2]) # Pto o L — every 2nd character # Other useful methods print(len(s)) # 14 — number of characters print(s.startswith('Py')) # True print(s.endswith('ML')) # True print(' '.join(['cat','dog','bird'])) # join a list into a string
1.6 — F-strings: The Professional Way to Format Output

F-strings let you embed variable values directly inside a string. They are the standard in all modern Python code. You will write thousands of f-strings in your ML career — for training logs, metric reporting, debugging, and UI output. Learn this format deeply.

An f-string starts with f before the quote. Variables go inside {curly braces}. You can also put format specifications after a colon inside the braces.

F-string formats — from basic to ML production level
name = 'Bilal' epoch = 15 total = 50 loss = 0.23412738 accuracy = 0.9423 salary = 85000 # Basic f-string print(f'Hello, {name}!') # Hello, Bilal! # Integer formatting print(f'Epoch {epoch} of {total}') # Epoch 15 of 50 print(f'Epoch {epoch:03d} of {total:03d}') # Epoch 015 of 050 ← zero-padded # Float formatting print(f'Loss: {loss}') # full precision — messy print(f'Loss: {loss:.4f}') # 0.2341 — 4 decimal places print(f'Loss: {loss:.2f}') # 0.23 — 2 decimal places # Percentage print(f'Accuracy: {accuracy:.1%}') # 94.2% ← auto × 100 print(f'Accuracy: {accuracy:.2%}') # 94.23% # Thousands separator print(f'Salary: Rs.{salary:,}') # Rs.85,000 # Width and alignment print(f"{'Metric':<15} {'Value':>10}") # aligned columns print(f"{'accuracy':<15} {accuracy:>10.4f}") print(f"{'loss':<15} {loss:>10.4f}") # Expression inside f-string print(f'Status: {"GOOD" if accuracy > 0.9 else "NEEDS WORK"}') # A real training log line: print(f'Epoch {epoch:03d}/{total:03d} | Loss: {loss:.4f} | Acc: {accuracy:.2%} | {"✓" if accuracy>0.9 else "✗"}')
1.7 — Operators: The Math of ML

Python supports all standard mathematical operations plus a few special ones that appear constantly in ML code. You need to know all of them.

OperatorNameExampleResultML Use
+Addition3 + 47Summing losses
-Subtraction10 - 37Error calculation
*Multiplication4 * 312Scaling values
/Division10 / 33.333Averages, ratios
//Floor division10 // 33Number of full batches
%Modulo (remainder)10 % 31Batch index cycling
**Power (exponent)2 ** 8256Squaring, square root
⭐
The two operators that appear most in ML math:
** — to square: error**2. To square root: x**0.5. To any power.
% — to check if a number divides evenly: if epoch % 10 == 0: save_checkpoint()
Operators in ML context
# Batch math dataset_size = 1000 batch_size = 32 full_batches = dataset_size // batch_size # 31 leftover = dataset_size % batch_size # 8 samples in last batch print(f'{full_batches} full batches + {leftover} leftover samples') # The power operator — used in loss functions error = 0.3 squared_error = error ** 2 # 0.09 — squaring the error print(f'Squared error: {squared_error}') # Square root using **0.5 mse = 0.09 rmse = mse ** 0.5 # 0.3 — square root print(f'RMSE: {rmse}') # MSE (Mean Squared Error) — your first ML formula # MSE = (1/n) × Σ(predicted - actual)² p1, a1 = 0.8, 1.0 # predicted=0.8, actual=1.0 p2, a2 = 0.3, 0.0 # predicted=0.3, actual=0.0 p3, a3 = 0.9, 1.0 # predicted=0.9, actual=1.0 mse = ((p1-a1)**2 + (p2-a2)**2 + (p3-a3)**2) / 3 print(f'MSE: {mse:.6f}') # You just implemented your first ML loss function! # Comparison operators — return True or False print(0.95 > 0.90) # True print(0.80 == 0.80) # True (use == not = for comparison) print(0.70 != 0.80) # True (not equal)
🏆
You just implemented MSE from scratch. Mean Squared Error is one of the most important loss functions in machine learning. You used only Chapter 1 Python — no libraries. This is exactly what libraries like sklearn calculate internally.
Chapter 1 Summary
  • Variables are named boxes. Python figures out the type automatically. Use snake_case naming.
  • 4 types: int (whole), float (decimal), str (text), bool (True/False). Each has a specific ML use case.
  • Type conversion: CSV files load everything as strings. Use int(), float(), str(). Never int() a decimal string directly — go through float() first.
  • Strings have methods: .strip(), .lower(), .split(), .replace(). Chain them for NLP preprocessing.
  • F-strings: f'Loss: {loss:.4f}'. Use {:03d} for zero-padded integers, {:.4f} for decimals, {:.2%} for percentages.
  • Operators: ** for power (squaring, sqrt), % for remainder (batch indexing), // for floor division (batch counting).
