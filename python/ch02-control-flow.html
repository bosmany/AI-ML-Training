Chapter 2:
Control Flow

A program that only runs top-to-bottom can't make decisions or repeat work. Control flow is what turns Chapter 1's static variables into something that behaves like a training loop: it decides, it repeats, it reacts. Every model you ever train runs inside a loop built from exactly the tools in this chapter.

🕐 ~50 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
2.1 — Boolean Logic: and, or, not

Every decision in code comes down to a value that is True or False. You already met comparisons (>, ==, ...) in Chapter 1 — Boolean logic is how you combine several of them into one decision.

OperatorMeaningExampleResult
andTrue only if BOTH sides are TrueTrue and FalseFalse
orTrue if AT LEAST ONE side is TrueTrue or FalseTrue
notFlips True ↔ Falsenot TrueFalse
Deciding if a model is ready to deploy
accuracy = 0.94 loss = 0.08 is_gpu_available = True print(accuracy > 0.9) # True print(loss < 0.1) # True print(accuracy > 0.9 and loss < 0.1) # True — BOTH must hold print(accuracy > 0.99 or loss < 0.1) # True — at least ONE holds print(not is_gpu_available) # False ready_to_deploy = accuracy > 0.9 and loss < 0.1 and is_gpu_available print(f'Ready to deploy: {ready_to_deploy}')
💡
Python reads left to right and stops early. In a and b, if a is already False, Python never even checks b — it already knows the answer is False. This is called short-circuiting and it's why x != 0 and 10/x > 1 is safe even when x could be zero.
2.2 — if / elif / else: Making Decisions

An if statement runs a block of code only when a condition is True. Chain elif ("else if") for more conditions, and finish with else for everything that didn't match. Python checks each condition top to bottom and stops at the first True one — order matters.

Grading a model's accuracy
accuracy = 0.83 if accuracy >= 0.95: grade = 'A — Production ready' elif accuracy >= 0.85: grade = 'B — Good, needs tuning' elif accuracy >= 0.70: grade = 'C — Needs more work' else: grade = 'F — Back to the drawing board' print(f'Accuracy {accuracy:.0%} → Grade: {grade}')
⚠️
Indentation is not style — it's syntax. Python uses indentation (4 spaces, consistently) to know what belongs inside the if block. Mixing tabs and spaces, or inconsistent indent width, causes an IndentationError.
Nested conditions — detecting overfitting
train_loss = 0.12 val_loss = 0.31 accuracy = 0.91 if accuracy > 0.9: if val_loss > train_loss * 1.5: verdict = 'High accuracy, but OVERFITTING — val loss much higher than train loss' else: verdict = 'High accuracy AND generalizing well' else: verdict = 'Accuracy too low — keep training' print(verdict)
2.3 — for Loops and range(): Repeating Work

A for loop runs a block once per item in a sequence. range(n) generates the numbers 0, 1, 2, ... n-1 — perfect for "do this N times," like running N training epochs.

for loops — over data, and over epochs
losses = [0.82, 0.61, 0.45, 0.30, 0.22, 0.18] # Loop directly over a list's values: for loss in losses: print(f'Loss: {loss:.2f}') # range(start, stop) — 'do this N times', typical epoch loop: for epoch in range(1, 6): print(f'Training epoch {epoch}') # range(start, stop, step) — every 10th epoch, e.g. for checkpoints for epoch in range(0, 50, 10): print(f'Checkpoint saved at epoch {epoch}')
2.4 — while Loops: Repeating Until a Condition Changes

A while loop keeps running as long as its condition stays True. Use it when you don't know in advance how many repetitions you need — like "keep training until the loss drops below a target," which is exactly how early stopping works.

Training until loss converges
loss = 0.9 epoch = 0 while loss > 0.1: epoch += 1 loss *= 0.7 # simulate loss shrinking 30% each epoch print(f'Epoch {epoch}: loss = {loss:.4f}') print(f'Converged after {epoch} epochs')
🛑
The #1 while-loop bug: forgetting to change the condition. If you write while loss > 0.1: and never update loss inside the loop, the condition is always True and your program hangs forever. Always double-check that something inside the loop moves you toward the exit.
2.5 — break and continue

continue skips the rest of the current iteration and jumps to the next one — useful for skipping bad or missing data. break exits the loop entirely — useful for stopping the moment you find what you're looking for.

continue to skip missing values, break to stop early
scores = [0.9, 0.85, None, 0.77, None, 0.95] # continue — skip missing (None) sensor readings valid_scores = [] for s in scores: if s is None: continue # skip this one, go to the next valid_scores.append(s) print('Valid scores:', valid_scores) # break — stop the moment we find a score below the 0.8 threshold for i, s in enumerate(valid_scores): if s < 0.8: print(f'First low score at index {i}: {s}') break # stop looking — we found it
2.6 — enumerate() and zip(): The Two Most-Used Loop Helpers in ML

enumerate(list) gives you both the index and the value as you loop. zip(list_a, list_b) walks two (or more) lists in parallel, pairing them up element by element. Together they are how you compare predictions against ground-truth labels.

Comparing predictions to targets
predictions = [1, 0, 1, 1, 0] targets = [1, 0, 0, 1, 0] # enumerate + zip: index AND both values, all at once for i, (p, t) in enumerate(zip(predictions, targets)): status = 'correct' if p == t else 'WRONG' print(f'Sample {i}: predicted={p} actual={t} → {status}') # zip() stops at the SHORTER list if lengths don't match — watch out for this correct = sum(1 for p, t in zip(predictions, targets) if p == t) accuracy = correct / len(targets) print(f'Accuracy: {accuracy:.2%}')
2.7 — List and Dict Comprehensions

A comprehension builds a new list (or dict) from an existing one, in a single readable line. It's the same idea as a for loop that appends to a list — just more compact, and it's the style you'll see constantly in real ML code before you ever touch NumPy.

Filter, transform, and map — the three comprehension patterns
raw_scores = [45, 78, 92, 33, 88, 61, 95] # Filter — keep only scores that pass (>= 60) passing = [s for s in raw_scores if s >= 60] print('Passing:', passing) # Transform — divide every score by 100 to normalize to 0-1 normalized = [s / 100 for s in raw_scores] print('Normalized:', normalized) # Filter + transform together passing_normalized = [s / 100 for s in raw_scores if s >= 60] print('Passing, normalized:', passing_normalized) # Dict comprehension — map each score to a pass/fail label grades = {s: ('pass' if s >= 60 else 'fail') for s in raw_scores} print(grades)
⭐
Read it as English: [s/100 for s in raw_scores if s >= 60] reads as "s divided by 100, for each s in raw_scores, if s is at least 60." Once you can read comprehensions this way, they stop feeling like a trick and start feeling obvious.
Chapter 2 Summary
  • and / or / not combine conditions. Python short-circuits — it stops evaluating as soon as the answer is known.
  • if / elif / else checks conditions top to bottom and runs only the first one that's True. Indentation defines the block.
  • for loops repeat over a sequence or over range(n). while loops repeat until a condition becomes False — always make sure that condition can actually change.
  • continue skips to the next iteration. break exits the loop immediately.
  • enumerate() gives you the index; zip() walks two lists together. Combine them to compare predictions against targets.
  • Comprehensions — [expr for x in list if condition] — build a new list in one line: filter, transform, or both.

🏗️ Mini Project: Early Stopping Trainer

Real training runs don't train for a fixed number of epochs blindly — they stop as soon as the model stops improving, to avoid wasting time and overfitting. You'll implement that logic yourself, over a fixed history of loss values, using only control flow.

📋 What to Build

  • Loop through the given loss_history list, one epoch at a time (use enumerate)
  • Track the best (lowest) loss seen so far
  • Track a "patience counter": if the current loss isn't at least min_delta better than the best, increase the counter; otherwise reset it to 0 and update the best
  • Print one log line per epoch showing the loss and the current patience counter
  • The moment the patience counter reaches the patience limit, print an "Early stopping" message with the best loss and the epoch it stopped at, then break out of the loop
  • If the loop finishes all epochs without triggering early stopping, print a "Completed all epochs" message instead
