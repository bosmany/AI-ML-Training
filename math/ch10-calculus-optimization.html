Chapter 10:
Calculus & Optimization

This is the Module 2 capstone — and the single idea that makes ML training possible: a derivative tells you which direction makes a function bigger or smaller. "Training a model" is nothing more than repeatedly asking "which way is downhill?" and taking a small step that way. By the end of this chapter, you will have implemented that algorithm — gradient descent — from scratch, and used it to discover a linear relationship in data with no library doing the learning for you.

🕐 ~60 min read 💻 5 exercises 🏗️ Module 2 Capstone Project 🧪 Quiz to complete
10.1 — The Derivative: Which Way Is Downhill?

The derivative of a function at a point tells you its slope there — how fast it's changing, and in which direction. If you're standing on a curve and the derivative is positive, going right increases the function's value; going left decreases it. That's the entire intuition you need: the derivative points uphill; the negative derivative points downhill.

10.2 — Computing a Derivative Numerically

You don't need to remember calculus rules to use derivatives in code. A numerical derivative approximates the slope by checking the function's value at two nearby points and dividing by the distance between them — a direct translation of "rise over run."

Numerical derivative — no calculus rules needed
def numerical_derivative(f, x, h=0.0001): return (f(x + h) - f(x - h)) / (2 * h) # rise over run, at a tiny scale def f(x): return x ** 2 for x in [-2, 0, 1, 3]: slope = numerical_derivative(f, x) print(f'At x={x}: slope ≈ {slope:.4f}') # Compare: the true derivative of x^2 is 2x — 2*(-2)=-4, 2*0=0, 2*1=2, 2*3=6. Matches!
💡
This is exactly what deep learning frameworks compute automatically — PyTorch's "autograd" and TensorFlow's "GradientTape" compute derivatives of enormous functions (entire neural networks) so you never have to derive them by hand. You're building the concept from the ground up here so that later, when a framework does it for you, you know exactly what it's doing.
10.3 — Gradient Descent: The Algorithm That Trains Every Model

To MINIMIZE a function, repeatedly step in the direction of the NEGATIVE derivative — downhill. This is gradient descent: the algorithm underneath linear regression, logistic regression, and every neural network you will ever train.

Minimizing (x-3)² — the minimum is obviously at x=3
def f(x): return (x - 3) ** 2 def gradient(x): return 2 * (x - 3) # the true derivative of (x-3)^2 x = 0.0 # start anywhere learning_rate = 0.1 # how big a step to take each time for step in range(10): grad = gradient(x) x = x - learning_rate * grad # step DOWNHILL — opposite the gradient print(f'Step {step+1}: x={x:.4f}, f(x)={f(x):.4f}') print(f'Approaching the true minimum at x=3: {x:.4f}')
10.4 — Learning Rate: The Most Important Number in ML Training

The learning rate controls step size. Too small, and training crawls. Too large, and you overshoot the minimum — sometimes so badly the loss explodes instead of shrinking.

Same problem, three learning rates
def f(x): return (x - 3) ** 2 def gradient(x): return 2 * (x - 3) for lr in [0.01, 0.5, 1.1]: x = 0.0 for step in range(5): x = x - lr * gradient(x) print(f'lr={lr} after 5 steps: x={x:.4f} (target: 3.0)') # lr=0.01: barely moved. lr=0.5: converges fast and cleanly. lr=1.1: OVERSHOOTS and diverges!
⚠️
If your model's loss is increasing or oscillating wildly during training, the learning rate is almost always the first thing to check. This single number causes more failed training runs than any other hyperparameter.
10.5 — Partial Derivatives: Gradients in Multiple Dimensions

Real models have many parameters, not just one. A partial derivative is the slope with respect to ONE variable, holding the others fixed. The gradient is the vector of all partial derivatives — it points in the direction of steepest increase, across every dimension at once.

Partial derivatives of f(x, y) = x² + y²
def f(x, y): return x**2 + y**2 def partial_x(x, y, h=0.0001): return (f(x+h, y) - f(x-h, y)) / (2*h) def partial_y(x, y, h=0.0001): return (f(x, y+h) - f(x, y-h)) / (2*h) x, y = 2, 3 print(f'∂f/∂x at (2,3): {partial_x(x, y):.4f}') # true value: 2x = 4 print(f'∂f/∂y at (2,3): {partial_y(x, y):.4f}') # true value: 2y = 6
Chapter 10 Summary
  • The derivative is slope — which way, and how steeply, a function is changing.
  • Gradient descent: repeatedly step in the direction of the negative gradient to minimize a function.
  • Learning rate controls step size — too small is slow, too large can diverge entirely.
  • Partial derivatives generalize this to functions of many variables — exactly what a model with many weights needs.
  • Training a model IS gradient descent on its loss function, with respect to its parameters. You just implemented the core loop.

🏗️ Module 2 Capstone: Linear Regression From Scratch

Everything converges here: Chapter 8's X @ w + b, Chapter 9's MSE, and this chapter's gradient descent. 📋 What to Build

  • Data: x = [1,2,3,4,5], y = [3,5,7,9,11] (the true relationship is y = 2x + 1)
  • Initialize w = 0.0, b = 0.0
  • Repeat 2000 times: compute predictions = w*x + b for every point, compute the MSE gradient with respect to w and b, and step both downhill
  • Gradient formulas: dw = (2/n) * sum(error * x), db = (2/n) * sum(error), where error = predictions - y
  • Print the learned w and b, and the final MSE — they should land close to the true w=2, b=1
