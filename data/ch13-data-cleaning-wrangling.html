Chapter 13:
Data Cleaning & Wrangling

Real datasets are never clean. Missing values, exact and near-duplicate rows, data spread across multiple files that need joining, and inconsistent text formatting — cleaning this up isn't a chore before "the real work," it IS most of the real work. Industry surveys consistently put data cleaning at 60-80% of a data scientist's time. This chapter is that 60-80%.

🕐 ~60 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
13.1 — Missing Values: Find Them, Then Decide

Missing data shows up as NaN (Not a Number) in pandas. .isna() flags every missing cell; .dropna() removes rows with any missing value; .fillna(value) replaces missing values — often with the column's mean, median, or a sentinel like 0 or 'unknown'.

Detecting and handling missing values
import pandas as pd import numpy as np df = pd.DataFrame({ 'name': ['Alice','Bob','Carol','Dave','Eve'], 'score': [92, np.nan, 81, 60, np.nan], 'city': ['Hubli', 'Pune', None, 'Delhi', 'Hubli'], }) print(df.isna().sum()) # missing count PER COLUMN print(f'Rows with any missing: {df.isna().any(axis=1).sum()}') filled = df.copy() filled['score'] = filled['score'].fillna(filled['score'].mean()) print(f'Filled mean score: {filled["score"].iloc[1]:.4f}') # Bob's NaN -> column mean dropped = df.dropna() print(f'Rows after dropna: {len(dropped)}') # rows with ANY NaN are gone
⚠️
dropna() vs fillna() is a real decision, not a default — dropping rows throws away every OTHER column's data too, just because one field was missing. Filling with the mean is safer for numeric columns you plan to keep using in bulk, but can quietly bias small datasets. There's no universally "correct" choice — know why you picked one.
13.2 — Duplicates: Exact Repeats

.duplicated() flags rows that are exact repeats of an earlier row (keeping the first occurrence as "not a duplicate"). .drop_duplicates() removes them.

Finding and removing duplicate rows
import pandas as pd df = pd.DataFrame({ 'name': ['Alice','Bob','Alice','Carol','Bob'], 'score': [92, 75, 92, 81, 75], }) print(f'Duplicate rows: {df.duplicated().sum()}') deduped = df.drop_duplicates() print(f'Rows after drop_duplicates: {len(deduped)}') print(f'Unique names: {list(deduped["name"])}')
13.3 — Merging: Combining Data Sideways

Real data lives in multiple tables that share a key column (like name or user_id). pd.merge(left, right, on='key', how=...) joins them, SQL-style. how='inner' keeps only keys present in BOTH tables; how='left' keeps every row from the left table, filling missing matches with NaN.

Inner vs left join
import pandas as pd candidates = pd.DataFrame({'name': ['Alice','Bob','Carol'], 'score': [92,75,81]}) cities = pd.DataFrame({'name': ['Alice','Bob','Dave'], 'city': ['Hubli','Pune','Delhi']}) inner = pd.merge(candidates, cities, on='name', how='inner') # Carol and Dave both dropped — no match in the other table left = pd.merge(candidates, cities, on='name', how='left') # Carol kept, city becomes NaN print(f'Inner join names: {list(inner["name"])}') print(f'Left join rows: {len(left)}')
⭐
Always check row counts after a merge — a merge key with duplicate values on either side silently multiplies rows (a "fan-out"), which is one of the most common real-world data bugs. If your merged table has more rows than you expected, that's almost always why.
13.4 — Concatenating: Stacking Data Vertically

Merging combines tables SIDEWAYS (matching columns by a key). pd.concat([df1, df2]) stacks tables ON TOP of each other — the classic "combine this week's export with last week's" operation. ignore_index=True renumbers the index so it doesn't repeat 0,1,2 twice.

Stacking two batches of data
import pandas as pd batch1 = pd.DataFrame({'name': ['Alice','Bob'], 'score': [92,75]}) batch2 = pd.DataFrame({'name': ['Carol','Dave'], 'score': [81,60]}) combined = pd.concat([batch1, batch2], ignore_index=True) print(f'Combined rows: {len(combined)}') print(f'Combined names: {list(combined["name"])}')
13.5 — Pivot Tables: Reshaping for Analysis

A pivot table reshapes long, row-per-observation data into a wide summary grid — one row per category, one column per another category, values aggregated at each intersection. This is the same mental model as an Excel pivot table.

Sales by rep, by quarter
import pandas as pd sales = pd.DataFrame({ 'rep': ['Alice','Alice','Bob','Bob','Carol','Carol'], 'quarter': ['Q1','Q2','Q1','Q2','Q1','Q2'], 'amount': [100, 120, 90, 95, 150, 140], }) pivot = sales.pivot_table(values='amount', index='rep', columns='quarter', aggfunc='sum') print(pivot) print(f'Alice Q2: {pivot.loc["Alice","Q2"]}') print(f'Bob total: {pivot.loc["Bob"].sum()}')
13.6 — Cleaning Text and Fixing Types

Real text data has stray whitespace and inconsistent casing (' Alice ', 'BOB', 'carol'). The .str accessor runs string methods across a whole column at once. And a column that LOOKS numeric but was loaded as text needs .astype() before any math will work correctly on it.

.str accessor + astype()
import pandas as pd messy = pd.DataFrame({'name': [' Alice ', 'BOB', 'carol'], 'score_text': ['92', '75', '81']}) cleaned = messy.copy() cleaned['name'] = cleaned['name'].str.strip().str.title() # trim whitespace, then 'Title Case' cleaned['score'] = cleaned['score_text'].astype(int) # text '92' -> real integer 92 print(list(cleaned['name'])) print(f'Total score: {cleaned["score"].sum()}') # only works because score is now numeric, not text
🚫
Summing a text column silently does string concatenation, not addition — '92' + '75' gives '9275', not 167, and pandas won't warn you. Always confirm a column's dtype (df.dtypes) before trusting arithmetic on it.
Chapter 13 Summary
  • .isna() / .dropna() / .fillna() find and handle missing values — dropping loses other columns' data, filling can bias the data. Choose deliberately.
  • .duplicated() / .drop_duplicates() find and remove exact repeat rows.
  • pd.merge(..., on=key, how=...) joins tables sideways by a shared key — check row counts for accidental fan-out.
  • pd.concat([...]) stacks tables of the same shape vertically.
  • .pivot_table() reshapes long data into a wide summary grid.
  • .str accessor + .astype() clean inconsistent text and fix wrong column types before doing any math.
