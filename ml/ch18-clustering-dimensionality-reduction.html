Chapter 18:
Clustering & Dimensionality Reduction

Every model so far has been supervised — trained on features AND a known correct answer. This chapter drops the answer. K-Means looks at unlabeled data and finds groups of similar points on its own — no y anywhere. PCA tackles a different problem: when you have many correlated features, it finds a smaller set of new features that captures most of the original signal — useful for visualization, noise reduction, and speeding up downstream models.

🕐 ~55 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
18.1 — Unsupervised Learning: No Labels

Everything in Chapters 15-17 was supervised: fit on (X, y) pairs, evaluated against known correct answers. Unsupervised learning only gets X — there IS no "correct" grouping to check against, only patterns in how the points are distributed. Clustering is the most common unsupervised task: group similar rows together without ever being told what the groups should be.

Twelve candidates, two features, no labels at all
import numpy as np names = ['Alice','Bob','Carol','Dave','Eve','Frank', 'Grace','Heidi','Ivan','Judy','Karl','Liam'] X = np.array([ [92,88],[75,70],[81,79],[60,65],[95,91],[55,60], [88,84],[70,68],[90,85],[58,62],[85,80],[65,63], ]) print(f'X shape: {X.shape}') print('There is no y — nothing here says which candidates are "similar"; we have to find out.')
18.2 — K-Means Clustering

KMeans(n_clusters=k) picks k group centers, assigns every point to its nearest center, moves each center to the average of its assigned points, and repeats until stable. After .fit(): .labels_ is the cluster ID for each training point, .cluster_centers_ are the k learned center points, and .inertia_ is the sum of squared distances from each point to its assigned center (lower = tighter clusters).

KMeans with k=2 on the candidate data
import numpy as np from sklearn.cluster import KMeans X = np.array([ [92,88],[75,70],[81,79],[60,65],[95,91],[55,60], [88,84],[70,68],[90,85],[58,62],[85,80],[65,63], ]) km = KMeans(n_clusters=2, random_state=42, n_init=10) km.fit(X) print(f'Labels: {km.labels_.tolist()}') print(f'Cluster centers: {[[round(c,4) for c in c2] for c2 in km.cluster_centers_]}') print(f'Inertia: {km.inertia_:.4f}')
⭐
K-Means found two clean groups without ever seeing a label — cluster 0 centers around (88.5, 84.5) (high scorers), cluster 1 around (63.83, 64.67) (lower scorers). Nothing told it "qualified" vs "not" — it discovered the split purely from how the points are distributed in space.
18.3 — Choosing K: The Elbow Method

Unlike supervised models, there's no accuracy to check — you have to CHOOSE k yourself. Inertia always decreases as k increases (more clusters = tighter fit, trivially — at k=n every point is its own cluster with 0 inertia). The elbow method plots inertia against k and looks for the point where adding more clusters stops giving a big improvement — the "elbow" in the curve.

Inertia across k=1 to k=4
import numpy as np from sklearn.cluster import KMeans X = np.array([ [92,88],[75,70],[81,79],[60,65],[95,91],[55,60], [88,84],[70,68],[90,85],[58,62],[85,80],[65,63], ]) for k in [1,2,3,4]: kk = KMeans(n_clusters=k, random_state=42, n_init=10) kk.fit(X) print(f'k={k}: inertia={kk.inertia_:.4f}')
📌
The biggest single drop is from k=1 to k=2 (3598.58 → 593.17) — after that, each extra cluster buys progressively less. That's the elbow: k=2 matches the two-group structure that's actually in this data, and this candidate dataset was in fact built from two natural score bands.
18.4 — Assigning New Points: .predict()

A fitted KMeans isn't just a one-time grouping of the training data — .predict() assigns brand-new points to whichever existing cluster center they're closest to, exactly like a supervised model's .predict(), just against learned CENTERS instead of a learned decision boundary.

Assigning two new candidates to existing clusters
import numpy as np from sklearn.cluster import KMeans X = np.array([ [92,88],[75,70],[81,79],[60,65],[95,91],[55,60], [88,84],[70,68],[90,85],[58,62],[85,80],[65,63], ]) km = KMeans(n_clusters=2, random_state=42, n_init=10) km.fit(X) new_candidates = np.array([[93,90], [57,59]]) new_labels = km.predict(new_candidates) print(f'New labels: {new_labels.tolist()}') # which existing cluster each new point joins
18.5 — Dimensionality Reduction: Why Fewer Features

Real datasets often have dozens or hundreds of features, many of them correlated (candidates who score well on one skill test tend to score well on related ones). More features means more noise, slower training, and features you can no longer visualize (nobody can plot 50 dimensions). Principal Component Analysis (PCA) finds new axes — combinations of the original features — ordered by how much of the data's variance (spread/information) they capture, so you can keep only the first few and still retain most of the signal.

⚠️
PCA needs scaled features first. It finds directions of maximum VARIANCE — if one feature ranges 0-100000 and another ranges 0-1, the first will dominate purely from its units, not because it's actually more informative. Always run StandardScaler before PCA, exactly like you will before any distance-based method in Chapter 19.
18.6 — PCA in sklearn

PCA(n_components=k).fit_transform(X) returns the data re-expressed in k new dimensions. .explained_variance_ratio_ tells you what fraction of the ORIGINAL variance each new dimension captures — summing these tells you how much information survived the compression.

Compressing 4 correlated score features down to 2
import numpy as np from sklearn.preprocessing import StandardScaler from sklearn.decomposition import PCA # interview, technical, communication, culture_fit — all correlated with overall performance X4 = np.array([ [92,88,90,85],[75,70,72,68],[81,79,80,77],[60,65,58,62], [95,91,93,89],[55,60,52,58],[88,84,86,82],[70,68,71,66], [90,85,88,83],[58,62,55,60],[85,80,83,78],[65,63,67,61], ]) scaler = StandardScaler() X4_scaled = scaler.fit_transform(X4) pca = PCA(n_components=2) X4_pca = pca.fit_transform(X4_scaled) print(f'Original shape: {X4.shape} -> Reduced shape: {X4_pca.shape}') print(f'explained_variance_ratio_: {[round(v,4) for v in pca.explained_variance_ratio_]}') print(f'Total variance kept: {sum(pca.explained_variance_ratio_):.4f}')
⭐
2 components keep 99.91% of the original 4 features' variance. Because these 4 scores were all correlated with one underlying "how good is this candidate" signal, almost everything collapses onto the first component alone (98.73%) — this is exactly the situation PCA is built for.
18.7 — Clustering After PCA

PCA and clustering combine naturally: reduce noisy, correlated, high-dimensional data down to its strongest few components, THEN cluster on the compressed representation — faster, and often more robust because the noisiest, least-informative directions were dropped before clustering ever saw them.

Same clusters, found on 2 PCA dimensions instead of 4 raw ones
import numpy as np from sklearn.preprocessing import StandardScaler from sklearn.decomposition import PCA from sklearn.cluster import KMeans X4 = np.array([ [92,88,90,85],[75,70,72,68],[81,79,80,77],[60,65,58,62], [95,91,93,89],[55,60,52,58],[88,84,86,82],[70,68,71,66], [90,85,88,83],[58,62,55,60],[85,80,83,78],[65,63,67,61], ]) X4_scaled = StandardScaler().fit_transform(X4) X4_pca = PCA(n_components=2).fit_transform(X4_scaled) km_pca = KMeans(n_clusters=2, random_state=42, n_init=10) km_pca.fit(X4_pca) km_orig = KMeans(n_clusters=2, random_state=42, n_init=10) km_orig.fit(X4_scaled) print(f'Labels on PCA data: {km_pca.labels_.tolist()}') print(f'Labels on scaled data: {km_orig.labels_.tolist()}') print(f'Same clusters: {(km_pca.labels_ == km_orig.labels_).all()}')
Chapter 18 Summary
  • Unsupervised learning works from X alone — no labels, no "correct answer" to check against.
  • KMeans(n_clusters=k).fit(X) groups similar points; .labels_, .cluster_centers_, and .inertia_ describe the result.
  • The elbow method picks k by looking for where adding clusters stops meaningfully reducing inertia.
  • .predict() assigns new points to the nearest already-learned cluster center.
  • PCA compresses correlated features into fewer components, ordered by how much variance each captures — always scale first.
  • PCA + clustering combine naturally: reduce dimensions first, then cluster on the compressed, less noisy representation.
