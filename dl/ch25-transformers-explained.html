Chapter 25:
Transformers Explained

Every modern LLM — GPT, Claude, Llama — is built from one core mechanism: attention. Chapter 24's RNNs process a sequence one step at a time, carrying a hidden state forward. Attention instead lets every position look directly at every other position, all at once, weighted by relevance — no sequential bottleneck, fully parallelizable, and (empirically) much better at long-range dependencies. This chapter builds real scaled dot-product attention with NumPy, from the ground up.

🕐 ~65 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
25.1 — Why Attention? Beyond Sequential Processing

An RNN reads a sequence left to right, one token at a time — token 100 can only "see" token 1 through 99 steps of a carried hidden state, which tends to fade over long distances (Chapter 24's vanishing gradient problem, again). Attention instead computes, for every token, a direct weighted combination of EVERY OTHER token in the sequence, in one matrix operation — no distance penalty, and every position can be computed in parallel instead of waiting for the previous step.

25.2 — Word Embeddings: Words as Vectors

Before any attention math happens, each word (or sub-word "token") is looked up in an embedding table — a matrix where row i is a learned vector representing token i. A sentence becomes a matrix: one row per token, d_model columns.

Looking up embeddings for a sentence
import numpy as np np.random.seed(0) vocab = ['the', 'cat', 'sat', 'mat'] d_model = 4 embedding_table = np.random.randn(len(vocab), d_model).round(2) # one learned row per vocab word sentence = ['the', 'cat', 'sat'] indices = [vocab.index(w) for w in sentence] X = embedding_table[indices] # fancy indexing: pulls out one row per word print(f'Indices: {indices}') print(f'X shape: {X.shape}') # (3, 4) — 3 words, 4-dim embedding each
25.3 — Query, Key, Value: The Three Roles

For every token, attention computes three vectors by multiplying its embedding through three learned weight matrices: a Query ("what am I looking for?"), a Key ("what do I offer?"), and a Value ("what do I actually contribute if attended to?"). A token's Query is compared against EVERY token's Key to decide how much attention to pay to each one.

VectorComputed asRole
Q (Query)X @ W_Q"What am I looking for?"
K (Key)X @ W_K"What do I offer, for others to match against?"
V (Value)X @ W_V"What I actually contribute once attended to"
25.4 — Scaled Dot-Product Attention

The full formula: Attention(Q,K,V) = softmax(Q @ K.T / sqrt(d_model)) @ V. Q @ K.T scores every query against every key (a similarity via dot product). Dividing by sqrt(d_model) keeps those scores from growing too large as dimensionality increases (which would make softmax too peaked/unstable). softmax turns each row of scores into a probability distribution — how much to attend to each other token. Multiplying by V produces the actual weighted output.

Full scaled dot-product attention
import numpy as np def softmax(x, axis=-1): e = np.exp(x - np.max(x, axis=axis, keepdims=True)) # subtract max for numerical stability return e / np.sum(e, axis=axis, keepdims=True) np.random.seed(42) d_model = 4 X = np.random.randn(3, d_model).round(2) # 3 tokens, 4-dim embeddings W_Q = np.random.randn(d_model, d_model).round(2) W_K = np.random.randn(d_model, d_model).round(2) W_V = np.random.randn(d_model, d_model).round(2) Q = X @ W_Q; K = X @ W_K; V = X @ W_V scores = Q @ K.T / np.sqrt(d_model) # shape (3, 3): every token's score against every token weights = softmax(scores, axis=-1) # each ROW sums to 1 — a distribution over "what to attend to" output = weights @ V # weighted combination of Values print(f'Scores shape: {scores.shape}') print(f'Each row of weights sums to: {[round(float(v),4) for v in weights.sum(axis=1)]}') print(f'Output shape: {output.shape}') # SAME shape as input — attention doesn't change dimensionality
⭐
The output shape always matches the input shape. Attention is a "mixing" operation — every output row is a weighted blend of ALL input rows' Values, but the shape (sequence length × d_model) never changes. That's what makes it stackable into deep transformer layers.
25.5 — Multi-Head Attention

Instead of one attention computation, real transformers run SEVERAL in parallel ("heads"), each with its own W_Q, W_K, W_V, each free to learn a different kind of relationship (one head might learn grammatical structure, another might learn topic similarity). The heads' outputs are concatenated and projected back to d_model — more expressive than a single attention computation, at roughly the same total compute.

25.6 — Positional Encoding: Attention Has No Sense of Order

Attention treats the input as a SET, not a sequence — shuffle the tokens and the attention math gives the same set of outputs, just reordered. That's a problem: "dog bites man" and "man bites dog" would look identical. Positional encoding adds a unique, deterministic sin/cos pattern to each position's embedding before attention runs, so position information survives.

Sinusoidal positional encoding
import numpy as np def positional_encoding(seq_len, d_model): pos = np.arange(seq_len)[:, None] i = np.arange(d_model)[None, :] angle_rates = 1 / np.power(10000, (2 * (i // 2)) / d_model) angles = pos * angle_rates pe = np.zeros((seq_len, d_model)) pe[:, 0::2] = np.sin(angles[:, 0::2]) # even dims: sine pe[:, 1::2] = np.cos(angles[:, 1::2]) # odd dims: cosine return pe pe = positional_encoding(4, 4) print(f'PE shape: {pe.shape}') print(f'PE[0]: {[round(float(v),4) for v in pe[0]]}') # position 0 is always [0, 1, 0, 1, ...] print(f'PE[1]: {[round(float(v),4) for v in pe[1]]}')

This positional encoding is simply ADDED to the token embeddings (X = token_embeddings + positional_encoding) before attention runs. Position 0 always gets the same pattern, position 1 always gets the same (different) pattern — deterministic and learned-model-independent.

Chapter 25 Summary
  • Attention lets every token look at every other token directly, in parallel — no sequential bottleneck like an RNN.
  • Query, Key, Value are three learned projections of each token's embedding (X @ W_Q, X @ W_K, X @ W_V).
  • Attention(Q,K,V) = softmax(Q @ K.T / sqrt(d_model)) @ V — the entire mechanism, one line.
  • Multi-head attention runs several attention computations in parallel for richer representations.
  • Positional encoding injects order information, since attention alone is order-blind.
