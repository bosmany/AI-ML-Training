Chapter 21:
Neural Network Fundamentals

Every deep learning framework — PyTorch, TensorFlow, JAX — is doing exactly what you're about to build by hand: a forward pass through layers of weighted sums and nonlinear activations, a loss that measures how wrong the output is, and backpropagation (the chain rule, mechanically applied) to compute how to nudge every weight to reduce that loss. Build it once from raw NumPy and PyTorch code stops looking like magic — it's this, with a GPU and better bookkeeping.

🕐 ~70 min read 💻 5 exercises 🏗️ 1 mini project 🧪 Quiz to complete
21.1 — What Is a Neural Network?

A neural network is layers of simple units stacked together. Each unit ("neuron") does exactly one thing: take inputs, multiply each by a learned weight, sum them plus a bias, then squash the result through a nonlinear activation function. Chapter 8 already taught you the matrix-multiply that computes an ENTIRE LAYER of neurons at once: X @ W + b. A "deep" network is just several of these layers chained together.

🧠
Why layers, why nonlinearity?
Stack linear layers with NO activation function between them and the whole stack collapses mathematically into one single linear layer — no extra power gained. The activation function between layers is what lets a network learn curved, non-linear decision boundaries (like XOR, which you'll train on in 21.5 — a single linear layer literally cannot solve XOR).
21.2 — Activation Functions

The sigmoid function squashes any real number into (0, 1) — useful for binary outputs (probabilities). ReLU (Rectified Linear Unit) just zeroes out negatives and passes positives through unchanged — simpler, faster, and the default choice inside hidden layers of modern networks.

Sigmoid and ReLU
import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) def relu(x): return np.maximum(0, x) z = np.array([-2.0, -0.5, 0.0, 0.5, 2.0]) print(f'Sigmoid: {[round(float(v),4) for v in sigmoid(z)]}') print(f'ReLU: {relu(z).tolist()}')
⭐
sigmoid(0) is always exactly 0.5 — the function is symmetric around the origin. This is a fast sanity check any time you implement it: if sigmoid(0) doesn't come out to 0.5, there's a bug in your formula.
21.3 — Forward Propagation: One Pass Through the Network

Forward propagation is just running the input through every layer in order: multiply by that layer's weights, add its bias, apply its activation, feed the result into the next layer. For a 2-layer network: a1 = sigmoid(X @ W1 + b1) then a2 = sigmoid(a1 @ W2 + b2) — a2 is the network's final prediction.

A full forward pass, 4 samples at once
import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) X = np.array([[0,0],[0,1],[1,0],[1,1]], dtype=float) # 4 samples, 2 features each W1 = np.array([[0.5, -0.3], [0.2, 0.8]]) # 2 inputs -> 2 hidden units b1 = np.array([0.1, -0.1]) W2 = np.array([[0.6], [-0.4]]) # 2 hidden units -> 1 output b2 = np.array([0.05]) z1 = X @ W1 + b1 a1 = sigmoid(z1) # hidden layer activations z2 = a1 @ W2 + b2 a2 = sigmoid(z2) # final predictions, shape (4, 1) print(f'Hidden activations shape: {a1.shape}') print(f'Output predictions: {[round(float(v),4) for v in a2.flatten()]}')

These predictions are near 0.5 and wrong (random initial weights) — that's expected. The network hasn't learned anything yet; that's what training does.

21.4 — Loss Functions: Measuring "How Wrong"

A loss function turns "predictions vs. true labels" into a single number to minimize. Mean Squared Error (MSE) — average of (true - predicted)² — works for regression. Binary cross-entropy is the standard choice for binary classification; it penalizes confident wrong predictions much more heavily than MSE does.

MSE and binary cross-entropy
import numpy as np y_true = np.array([0, 1, 1, 0], dtype=float) y_pred = np.array([0.1, 0.9, 0.8, 0.3]) mse = np.mean((y_true - y_pred)**2) print(f'MSE: {mse:.4f}') eps = 1e-9 # avoids log(0) bce = -np.mean(y_true*np.log(y_pred+eps) + (1-y_true)*np.log(1-y_pred+eps)) print(f'Binary cross-entropy: {bce:.4f}')
21.5 — Backpropagation: The Chain Rule, Mechanically

Backpropagation computes the gradient of the loss with respect to EVERY weight, by applying the chain rule backward from the output. For a 2-layer network with sigmoid activations and MSE loss, the gradients have a clean closed form (derived once, reused forever):

QuantityFormula
d_a2 (output error)(a2 - y_true) * 2 / n_samples
d_z2d_a2 * a2 * (1 - a2) # sigmoid derivative: a*(1-a)
d_W2, d_b2a1.T @ d_z2,   d_z2.sum(axis=0)
d_a1d_z2 @ W2.T
d_z1, d_W1, d_b1same pattern, one layer earlier
One backprop step, by hand
import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) np.random.seed(7) X = np.array([[1.0, 0.5]]) y = np.array([[1.0]]) W1 = np.array([[0.2, -0.1],[0.3, 0.4]]) b1 = np.zeros(2) W2 = np.array([[0.5],[-0.2]]) b2 = np.zeros(1) # forward z1 = X @ W1 + b1; a1 = sigmoid(z1) z2 = a1 @ W2 + b2; a2 = sigmoid(z2) loss_before = np.mean((y - a2)**2) # backward d_a2 = (a2 - y) * 2 / len(X) d_z2 = d_a2 * a2 * (1 - a2) d_W2 = a1.T @ d_z2 d_b2 = d_z2.sum(axis=0) d_a1 = d_z2 @ W2.T d_z1 = d_a1 * a1 * (1 - a1) d_W1 = X.T @ d_z1 d_b1 = d_z1.sum(axis=0) # gradient descent step (Chapter 10!) lr = 1.0 W2 = W2 - lr*d_W2; b2 = b2 - lr*d_b2 W1 = W1 - lr*d_W1; b1 = b1 - lr*d_b1 # forward again, after the update z1 = X @ W1 + b1; a1 = sigmoid(z1) z2 = a1 @ W2 + b2; a2 = sigmoid(z2) loss_after = np.mean((y - a2)**2) print(f'Loss before: {loss_before:.4f}') print(f'Loss after one step: {loss_after:.4f}')
⭐
One gradient step, measurably lower loss. That's the entire training loop, over and over: forward pass → compute loss → backward pass → update weights → repeat. PyTorch's loss.backward() and optimizer.step() (Chapter 22) do exactly this, automatically, for networks with millions of weights.
21.6 — The Training Loop: Solving XOR

XOR is famous in deep learning history because it's NOT linearly separable — no single straight line divides the "true" points from the "false" points, so a single-layer network (no hidden layer) provably cannot solve it. A 2-layer network can. Watch the loss fall over 5000 epochs of forward → backward → update.

Training a 2-layer network on XOR
import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) np.random.seed(1) X = np.array([[0,0],[0,1],[1,0],[1,1]], dtype=float) y = np.array([[0],[1],[1],[0]], dtype=float) # XOR truth table W1 = np.random.randn(2, 4) * 0.5; b1 = np.zeros(4) W2 = np.random.randn(4, 1) * 0.5; b2 = np.zeros(1) lr = 0.5 for epoch in range(5000): a1 = sigmoid(X @ W1 + b1) a2 = sigmoid(a1 @ W2 + b2) loss = np.mean((y - a2)**2) d_a2 = (a2 - y) * 2 / len(X) d_z2 = d_a2 * a2 * (1 - a2) d_W2 = a1.T @ d_z2; d_b2 = d_z2.sum(axis=0) d_a1 = d_z2 @ W2.T d_z1 = d_a1 * a1 * (1 - a1) d_W1 = X.T @ d_z1; d_b1 = d_z1.sum(axis=0) W2 -= lr*d_W2; b2 -= lr*d_b2 W1 -= lr*d_W1; b1 -= lr*d_b1 print(f'Final loss: {loss:.4f}') print(f'Rounded predictions: {[int(round(float(v))) for v in a2.flatten()]}') # [0, 1, 1, 0]
Chapter 21 Summary
  • A layer is activation(X @ W + b). A network is several layers chained together.
  • sigmoid squashes to (0,1) for probabilities; ReLU is the default hidden-layer activation.
  • Forward pass computes predictions. Loss (MSE or cross-entropy) measures how wrong they are.
  • Backpropagation uses the chain rule to compute every weight's gradient. Gradient descent (Chapter 10) uses those gradients to improve the weights.
  • Repeat forward → loss → backward → update, and a network with a hidden layer can learn non-linear patterns like XOR that a single layer cannot.
